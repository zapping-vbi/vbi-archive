Requirements

  • An analog TV card with V4L, V4L2 or XVideo driver. Digital TV (DVB, ATSC) is not supported yet.
  • A Gnome 2.0 or later desktop, with the common libraries.
  • For Teletext and Closed Caption functions the zvbi library. VBI capturing is only possible on Linux and FreeBSD. It makes little sense to install the library on other Unices.
  • The video recording plugin needs the rte library, also available from the Zapping dowload page. You need not download mp1e, this codec is already included in the rte library. Presently recording is only supported on Linux.
  • The remote control plugin needs the LIRC package. If not already part of your distribution you can download LIRC from http://www.lirc.org. Presently remote controls are only supported on Linux.

Binaries

If possible you should use binaries which have been created for your distribution.

RedHat

RedHat RPMs are available from http://freshrpms.net/.

Debian

Debian packages are available from Debian mirrors. To install Zapping type

apt-get install zapping

Anonymous CVS Access

To get the latest development version of Zapping you can access the CVS server anonymously. If you are unfamiliar with using the CVS utility and the CVS commands see the CVS info or manual pages or go to the CVS web site for further information.

Step 1: Set the $CVSROOT environment variable. For sh/ksh/bash:

export CVSROOT=:pserver:anonymous@cvs.sf.net:/cvsroot/zapping

For csh/tcsh:

setenv CVSROOT :pserver:anonymous@cvs.sf.net:/cvsroot/zapping

If you don't want to set $CVSROOT you can use the cvs -d option:

cvs -d :pserver:anonymous@cvs.sf.net:/cvsroot/zapping login etc

Step 2: To log in to the Zapping CVS server type the following. When asked for a password just press enter.

cvs login

Step 3: To download the latest version of Zapping type:

cvs co zapping

You can request another version or development branch using the co -r TAG MODULE notation. This command for example requests Zapping version 0.7:

cvs co -r zapping-0-7 zapping

Other modules besides zapping are vbi containing the zvbi library and rte containing the rte library and mp1e. To find out which tags are available you can use the cvs status command or browse the CVS repository. Select a module and open the tag menu at the bottom of the page.

From now on you can type

cvs update -d -P

in the respective source directory to update your copy of the CVS repository. As before, -r will choose a particular version or branch and -A the latest version.

Step 4: Prior to the first compilation run

./autogen.sh --prefix=/usr

to create missing configuration files. Change the install prefix as needed. The autogen.sh script requires several GNU build tools like autoconf or libtool, will tell you if any are missing or too old.

See the Compile and Install page for further instructions.
